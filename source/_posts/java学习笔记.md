## java 学习笔记

#### day 1

1. java 的 Boolean 类型只有 true 和 false 不能用 0 和 1 替换

2. 多种数据混合运算时自动转换成精度最大的计算

3. (byte, short)和 char 之间不会相互自动转换

4. byte, short, char 三者计算时自动转换成 int

   同样这三者只要出现一个则他们的结果自动变为 int 类型

5. java 浮点数默认为 double

6. char 类型可以保存 int 常量值，但不能保存 int 变量值，需要强制转换

7. 基本数据类型转换为 String 只需要 + “ ”

String 转基本数据类型通过包装类调用 parseXX 方法即可

即使用基本数据类型对应的包装类的相应方法得到基本数据类型

```java
    e.g.  String s1 = "123";

​          int num1 =  Integer.parseInt(s1);
```

#### day 2

1.取模运算本质是 a % b = a - a / b \* b

e.g. 10 % -3 = 10 - 10 / (-3) \* -3 = 10 - 9 = 1

#### day 3

<strong>进制转换</strong>

1.十进制转换二/八/十六进制

不断除相应的进制数，将得到的余数倒过来，就是对应的进制。

2.二进制转八进制

将二进制数每三个一组转换成相应的八进制数即可。

3.二进制转十六进制

将二进制数每四个一组转换即可。

4.八进制或十六进制转换成二进制 2.和 3.倒过来同理。

5.\>> \<<算数位移

低位溢出，符号位补位溢出的高位。

左移几位相当于\*了几个 2，右移几位相当于/了几个 2。

\>>>逻辑位移

低位溢出，高位补零

##### 注意：没有<<<位移符

#### day 4

1.switch 表达式中的返回值必须是:byte，short，int，char，enum，String

没有 double 和 float 返回值。

2.case 语句中的值必须为常量。

3.如果 switch 语句没有 break 会发生穿透，会继续执行后边的 case 无视判断。

4.do while 是先执行后判断，注意 do while 语句在 while 的后面有个分号。

#### day 5

1.声明数组有两种方式

(1)数据类型 数组名[ ] e.g. int a[]

(2)数据类型[ ] 数组名 e.g. int[] a

2.创建数组

数组名 = new 数据类型[大小]

e.g. a = new int[10]

3.数组是多个相同类型的元素的集合，注意要类型相同 。

4.基本数据类型赋值为值拷贝 只是传递值。

5.数组默认情况下是引用传递 传递的是地址 arr2 的变化会影响到 arr1。

#### day 6

1.方法定义时的参数叫形式参数，简称形参。方法调用时的参数叫实际参数，简称实参。

形参和实参类型要一直或兼容，个数，顺序必须一致。

2.同一个类中的方法直接调用即可，跨类调用方法要通过对象名调用。

3.基本类型数据的传递为值传递，形参改变不影响实参。

引用类型数据的传递传递的是地址（传递的也是值，但值是地址），可以通过形参影响实参。

4.方法重载要求方法名和形参列表不同，对返回值无要求。

5.可以通过...来使用可变参数，可变参数可以当做数组来使用，比如 num.length。

可变参数的实参可以传入数组，本质也是数组。

可变参数可以和普通类型参数放在形参列表，但可变参数要放到最后。

#### day 7

1.全局变量可以不赋值有默认值，局部变量没有默认值使用必须赋值后才可使用。

2.属性和局部变量可以重名，访问时采用就近原则。

3.在同一个作用域中，两个局部变量不能重名。

4.全局变量可以加修饰符，局部变量不可加修饰符。

5.构造器修饰符可以默认，构造器没有返回值，方法名和类名一定要一样。

6.一旦定义了自己的构造器就不会再默认创建无参构造器，除非自己显式定义。

#### day 8

1.this 是创建对象时自带的属性，指向自身，用来简化构造器初始化属性。

2.this 可以访问成员方法，比如 this.方法名(参数列表)。

3.this 可以访问构造器，但只能在构造器中使用。比如 this(参数列表)。同样，用 this 访问其他构造器必须置于第一行语句。

4.this 只能在类定义的内部使用，不能在外部使用。

5.this 可以访问本类的属性。

#### day 10

1.子类继承了所有属性和方法，但私有属性和方法不能在子类直接访问，要通过父类提供的方法间接访问。

2.子类必须调用父类的构造器，完成父类的初始化。

3.创建子类对象时不管子类使用哪个构造器，默认情况下都调用父类无参构造器，如果父类没有无参构造器，则使用 super 指定使用父类的哪个构造器完成父类初始化，否则编译不通过。

4.一个对象编译类型和运行类型可以不一致。

5.编译类型在定义对象时就确定了不能改变，运行类型是可以变化的。编译类型看定义时 = 的左边，运行类型看 = 的右边。

6.**向上转型** 父类引用名 = new 子类类型();

可以调用父类中的所有成员，但不能调用子类中的特有成员。在编译阶段能调用那些成员是由编译类型来确定。

最终运行效果看子类具体实现，即调用方法时从子类开始找方法然后调用。

7.**向下转型** 子类类型 引用名 = (子类类型) 父类引用;

只能强转父类的引用，不能强转父类的对象。要求父类引用必须指向当前目标类型的对象。

向下转型后可以调用子类类型中的所有成员。

8.instanceof 为比较操作符，用于判断对象的运行类型是否为 XX 类型或者 XX 类型的子类型。

#### day 11

1.java 动态绑定机制：当调用对象方法的时候，该方法会和该对象的内存地址/运行类型绑定；当调用对象属性时，没有动态绑定机制，哪里声明哪里使用。

2.==判断基本数据类型为判断值是否相等，判断引用数据类型判断地址是否相等。

3.equals 默认判断引用数据类型地址是否相等，在子类中往往重写该方法用于判断内容是否相等。

4.静态方法可以直接通过类名.方法名调用。

5.static 变量是同一个类所有对象共享，在类加载的时候就生成了。

6.类方法中无 this 参数，普通方法中隐含 this 参数。

7.静态方法 main 可以访问本类的静态成员和方法，不可以访问本类的非静态成员和方法。如果想访问，创建一个对象实例调用即可。

8.static 代码块作用是对类初始化，随着类的加载而执行，且只会执行一次。普通代码块每创建对象就执行一次。

#### day 12

1.java 构造器隐含了执行要求第一步 super()；第二步调用普通代码块，随后才是执行构造步骤。

2.final 类不能继承但是可以实例化对象。如果不是 final 类但含有 final 方法，该方法不能被重写但是可以被继承。
